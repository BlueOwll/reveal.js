Regular expressions are very popular instrument and it is used for text parsing, searching and replacing data and other things. They are supported by most of programming languages and of course by Javascript. It is common to use it handling HTMLinner code, extracting class names, validating input. It seems simple but it takes some time to feel free using it.

What is regular expression? It’s a pattern, that is a row of characters, special characters and operators.
What is regular expression? 

						/hello/g
==> hello
They are specified separately (after regexp itself, or as separate argument of function) and show how to search. they are:

Flags

  • g - global search - all matchings (by default it's "local", we find only the first matching)
  • i - case in-sensitive
  • m - multiline search
    • .. and other:Flags
Every character in regular expresiion means ONE character in matching string. All characters in regular expression as a, b, ! etc must appear literally.

Matching

Hello! ==> Hello!

each character means ONE character
The set of characters can be specified using square brackets: [abc] match a, or b, or c You can add range as [a-z] and punctuation marks (but be sure that - hyphen is in the beginnig or in the end of set to avoid turning to a range) If you mention the caret at the beginning of set it will mean that there should be any character excluding specified. [^abc] => 2 or t or ? In square brackets all characters excluding^ mean literally itself.

Alternatives

[abc] ==> match a or b or c

Ranges
[a-zA-Z.&?] ==> any ONE latin letter or punctuation marks

Exclusion
[^abc] ==> any, but a, b, c, e.g. 2 or t or ?

You can specify how much time the character should appear (or group of caracters)

Quantifiers

  • ? - from 0 to 1
  • * - from 0 to many
  • + - from 1 to many
    • and more strict:
  • {2} - 2 times exactly
    {2,6} - from 2 to 6
But in the rest expression you should use escape sign backslash to show that this is not operator.

Escape character

\[  \.  \?  \\


Other useful special characters

. - any character
^ - word beginning
$ - word end
The most interesting and complicated opportunities are provided by parEntheses. Firstly they allow to group a sequence of characters to tell how much time they shoul appear. Usually the quantifier corresponds to one character.

Parentheses

group characters for repeating:
(ab)+ ==> ab or abab or ababababab
Secondly they form capturing groups. For example file names: (\w+)\.(\w+) In this case we can pick out only the file name (group #1) and only the extention (group #2).

Parentheses

group characters for capturing and handling:
(\w+)\.(\w+) ==>
matching example.txt with group#1 example and group#2 txt
We can refer to the group in the same expression: Here we tell that exactly the same expression must repeat.
Refer to group in the same expression

(\d\d)\1+ ==> 121212 or 9494
We can nest groups and refer to each of them: outer parentheses is group # 1 - ful file name inner groups 2 and 3 - file name and extension.
Nested group

file:\s+((\w+)\.(\w+)) ==>
matching file: example.txt
with captured group#1 example.txt, group#2 example and group#3 txt
There a lot of other opportunities, that make the regular expression extremely powerfull but quite tricky for understanding and using. And JavaScript has some special methods to use these opportinites as much as possible. You can construct regexp as a literal or as an object.

Regular Expression in JavaScript


						const regex1 = /\w+/;
						const regex2 = new RegExp('\\w+');
(the regexp as a string, so it’s important to use escape sign for backslash)
Regexp is used when manipulating strings (using String.prototype methods):

Manipulating strings (using String.prototype methods)


							str.match(regexp);
if flag g: returns array of matching without groups
in other case returns array:
full matching, capture groups, position in input string and the input string

Manipulating strings (using String.prototype methods)


							str.matchAll(regexp)
if flag g: returns arraylike object of arrays:
full matching, capture groups, position in input string and the input string
Manipulating strings (using String.prototype methods)


							str.split(regexp)
splits the string on every matching expression
Manipulating strings (using String.prototype methods)


							str.search(regexp)
returns the position of first matching (or -1 if nothing found)
Manipulating strings (using String.prototype methods)


							str.replace(str|regexp, str|func)
replace all (if there is flag g) matching substrings with string or using function,
that accepts array with data of matching including groups and position(spread of:
full matching, capture groups, position in input string and the input string)
The following methods are regexp object methods, and they execute iterative seaching (if there is flag g). After each call they fix the lastIndex and next time begin the searching from that position. we can specify this index to begin searching from particular point

Using RegExp.prototype methods

						regexp.exec(str)
